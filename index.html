Unit 2: Computer Systems and Networks – Review Vocabulary Computing Device: A physical device that can run a program (e.g., computer, smartphone, smart sensor). Computer Network: A group of interconnected computing devices capable of sending or receiving data. Path: The series of connections between computing devices on a network, starting with the sender and ending with the receiver. Protocol: An agreed-upon set of rules that specify how systems behave and communicate. Fault Tolerant: A system’s ability to continue functioning even if individual components fail. Scalability: The ability of a system to expand or contract to meet new demands. Bandwidth: The maximum transmission capacity of a system, measured by bit rate (bits per second). URL (Uniform Resource Locator): An easy-to-remember web address (e.g., www.example.com). Internet Structure and Layers Physical Network: The hardware and physical connections that carry internet signals. Internet Protocol (IP): Assigns unique IP addresses to devices and determines how data is routed between them. Transmission Control Protocol (TCP): Ensures reliable, ordered, and error-checked delivery of data packets. User Datagram Protocol (UDP): A lightweight alternative to TCP with minimal error-checking (faster but less reliable). Packets: Small chunks of data into which larger messages are divided for transmission. Packets may arrive in order, out-of-order, or not at all. Routers: Specialized computers that forward data across networks, choosing the fastest available path. Domain Name System (DNS): Translates human-readable domain names (like example.com) into IP addresses. Hypertext Transfer Protocol (HTTP): Used for transmitting and requesting web pages and media on the World Wide Web. World Wide Web (WWW): A system of linked pages, media, and files browsable over HTTP. Key Concepts Message Delivery on the Internet: Messages are split into packets, which may take different paths through the network. The internet does not require a direct connection between sender and receiver. Redundancy: Multiple pathways exist in the network so if one route fails, packets can take another. Fault Tolerance: Because of redundancy, the internet can continue working even if parts of it are damaged. Scalability: The internet is designed to grow and support billions of devices without redesigning its core structure.
